## 2. Design tokens

Tokens are CSS custom properties (per the FE proposal — the durable layer both Tailwind `@theme` and
raw component CSS read from). Values below are **provisional, read from the 6d wires**, and are
**reconciled with the hi-fi export** (which is authoritative, and introduces the third "surprise"
palette + full polish).

### Colour *(provisional — from 6d; finalized from hi-fi)*

| Token (provisional) | Value | Used for |
|---------------------|-------|----------|
| `--color-bg` | `#ffffff` | content background |
| `--color-bg-tint` | `#f7f5f2` | subtle section fill (e.g. Contribute) |
| `--color-fg` | `#2b302f` | primary text (body/lyrics) |
| `--color-fg-strong` | `#1a1a1a` | headings |
| `--color-muted` | `#5e6b68` | secondary text / labels (Corduroy) |
| `--color-meta` | `#8a8a86` | dates, fine print |
| `--color-accent` | `#7a2432` | primary actions, active nav, section labels (oxblood) |
| `--color-line` | `#e6e6e3` | hairlines / dividers |

> Note: 6d leaned the **muted-highway lane** into **Corduroy `#5e6b68` + an oxblood accent
> `#7a2432`** with warm neutrals — the Periwinkle/Pumice/Polo-Blue from the extracted palette are not
> prominent. The hi-fi may adjust this and adds the third palette.
>
> **Contrast is contractual (NFR-1):** body text ≥ 4.5:1, large text/UI ≥ 3:1. Every finalized token
> pairing must pass.

### Typography *(families locked; scale from hi-fi)*

| Token | Family | Used for |
|-------|--------|----------|
| `--font-display` | **Oswald** (400–700) | wordmark, headings, nav, labels, buttons |
| `--font-body` | **Georgia** (serif) | body copy, **lyrics**, notes, quotes |
| `--font-meta` | system-ui sans | dates, counts, fine print |

> (The *Kalam* cursive in the canvas is Claude Design's annotation font, **not** a site token.) The
> full type scale (sizes/line-heights/weights) is taken *(from hi-fi)*.

### Spacing, radius, elevation *(from hi-fi)*

- **Spacing scale:** *(from hi-fi — 6d reads roughly as a 4/8-based scale.)*
- **Radius:** small (~2–4px on cards/controls) *(confirm from hi-fi)*.
- **Elevation:** subtle card shadow + a sticky/scrolled header shadow *(confirm from hi-fi)*.

## 4. Visual states & patterns

How the wireframe state matrix *looks* (treatments *from hi-fi*, informed by 6d):

- **Empty (landing no-results, FR-19):** explicit message + reset; not a blank area.
- **Loading:** server-rendered; unobtrusive (no heavy skeletons for slice 1).
- **Error:** the site's standard error page; inline messages use `--color-accent` sparingly.
- **Focus / keyboard (NFR-1):** a visible focus ring on every interactive element; logical tab order.
- **Motion:** minimal (header solidify on scroll; hover transitions) — durations/easing *(from hi-fi)*.

This document is the single source of truth for **where the project is**. It is extracted from the
project brief so the brief stays stable after approval while this tracker is updated as each milestone
completes. The skills keep it honest: `sfk-version` lays down a version's milestone table from its
goals; `sfk-next-milestone` marks the next milestone `In progress` and commits its draft deliverable;
`sfk-signoff` flips it to `Complete` and moves the *Current position* line. Tracker and work move
together, commit by commit.

## Conventions

- **Status values:** ‚¨ú Not started ¬∑ üî∂ In progress ¬∑ ‚úÖ Complete
- A milestone is marked **In progress** (üî∂) when work on it starts (`sfk-next-milestone`), and
  **Complete** (‚úÖ) only after **explicit human sign-off** via `sfk-signoff` ‚Äî not merely when its
  deliverable looks finished (see root `CLAUDE.md`, *Milestone status lifecycle*).
- Each milestone is conducted in its own session in the assigned tool, with the relevant prior
  documents available as context.
- New versions are started with `sfk-version` (a version brief + that version's milestone table) and
  appended here only once their goals are agreed.
